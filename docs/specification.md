## 1. システム概要

### 技術構成

| 項目 | 技術 |
|------|------|
| フロントエンド | Next.js (React) |
| バックエンド | Express.js (Node.js) |
| データベース | MySQL (Railway) |
| ホスティング | Railway (自動デプロイ) |
| AI | OpenAI GPT (通話評価・分析) |
| 外部連携 | Google Sheets API (文字起こしログ) |

### 認証方式

- オペレーター: 名前選択でログイン（パスワードなし）
- 管理者/マネージャー/営業: メールアドレス + パスワード
- JWT トークンによるセッション管理
- ログイン時にlocalStorageへトークン保存（端末ごとに独立）

## 3. ダッシュボード

**画面:** `/` (トップページ)
**権限:** 全ロール

### 3.1 KPI カード

上部の期間タブ（日別/週別/月別/累計）で全KPIとグラフが連動して切り替わる。

| KPI | 説明 | 補助情報 |
|-----|------|---------|
| 稼働時間 | 手動入力の勤務時間（クリックで入力モーダル） | 平均h/日 |
| コール数 | 期間内の総架電数（SKIP除外） | /h |
| リコール獲得 | リコール設定した件数 | /h |
| リコール消化 | 完了したリコール件数 | /h |
| 有効接続 | 相手と会話できた件数 | /h |
| 担当接続 | 担当者と会話できた件数 | /h |
| 案件獲得 | 案件化した件数 | h/件（獲得効率） |

### 3.2 スコープ切替（管理者/マネージャーのみ）

- **全体:** チーム全員の合算値
- **オペレーター別:** 個人を選択して表示

### 3.3 グラフ

| グラフ | 内容 | 表示形式 |
|--------|------|---------|
| 時間帯別コール数 | 9時〜19時の各時間帯のコール数 | 棒グラフ |
| 業種別案件化率 | 案件化コールの業種別構成比 | 円グラフ |
| 時間帯×業種別 接続数/接続率 | 時間帯と業種のクロス集計 | テーブル |

全グラフはメインの期間タブに連動して表示が切り替わる。

### 3.4 コピー機能

| ボタン | 内容 |
|--------|------|
| コールデータ | KPI値を改行区切りでクリップボードにコピー |
| 日報コピー | 「コール時間：XX」形式の日報テキストをコピー |

### 3.5 AI総合分析（管理者/マネージャーのみ）

- チーム全体 or 個人のパフォーマンスをAIが分析
- 期間指定（日別/週別/月別/累計）
- AIコーチング（個人選択時）

## 4. 架電画面

**画面:** `/call`
**権限:** operator, manager, admin

### 4.1 架電リスト

左サイドバーに最大10件の架電対象を表示。ピックアップモード：

| モード | 説明 |
|--------|------|
| **自動** | 優先順位に基づいて自動選択（デフォルト） |
| **業種別** | 業種を選択してフィルタリング |
| **自作** | 自作リストからピックアップ |
| **特別** | 特別リストからピックアップ |

**優先順位ロジック:**
1. 優先オペレーター割当（猶予日数内）
2. 架電回数が少ない企業を優先
3. 最終架電からの経過日数
4. NG企業・既存案件リスト除外

### 4.2 企業情報表示

- 会社名（クリックで「会社名 ハローワーク」Google検索）
- 電話番号、業種、地域、住所
- 過去の架電履歴（結果コード、メモ、日時）

### 4.3 架電結果入力

| 結果コード | 説明 |
|-----------|------|
| NO_ANSWER | 不通 |
| NG | 不在/拒否 |
| RECALL | リコール（日時指定） |
| INTERESTED | 興味あり |
| PROJECT | 案件化 |
| SKIP | スキップ（集計対象外） |

**入力項目:**
- 結果コード（必須）
- メモ（任意）
- 有効接続チェック
- 担当接続チェック
- リコール日時（RECALL選択時）

### 4.4 スクリプトパネル

- 切り返しトーク・Q&Aを検索/閲覧
- 業種フィルタリング対応

### 4.5 ロック機構

- 企業単位で排他ロック（5分タイムアウト）
- 他のオペレーターが架電中の企業は表示されない

## 5. リコール管理

**画面:** `/recalls`
**権限:** operator, manager, admin

### タブ分類

| タブ | 対象 |
|------|------|
| 今日 | 本日期限のリコール |
| 明日 | 翌日期限のリコール |
| 期限超過 | 期限切れの未完了リコール |
| その他 | 上記以外の予定リコール |

### 操作

- **完了:** リコールを消化済みにする
- **キャンセル:** リコールを取消す
- 通話文字起こしの閲覧
- クリックで架電画面へ遷移

## 6. 案件管理

**画面:** `/projects`, `/projects/[id]`
**権限:** 全ロール（営業は閲覧中心）

### 6.1 案件一覧

- ステータスフィルタ、日付範囲フィルタ、自分の案件フィルタ
- **インライン ステータス変更:** 一覧上でプルダウンから直接ステータスを変更可能
- 案件クリックで詳細画面へ遷移

### 6.2 ステータス一覧

| ステータス | 説明 |
|-----------|------|
| BOSHUCHU | 募集中 |
| SHORUI_CHU | 書類選考中 |
| SHORUI_OCHI | 書類落ち |
| MENSETSU_KAKUTEI | 面接確定 |
| KEKKA_MACHI | 結果待ち |
| NAITEI | 内定 |
| NAITEI_TORIKESHI | 内定取消 |
| FUGOKAKU | 不合格 |
| LOST | 失注 |
| BARASHI | バラシ |
| HORYU | 保留 |
| KISON_NASHI | 既存なし |
| MODOSHI | 戻し |
| MODORI | 戻り |

### 6.3 案件詳細

- 企業情報の表示/編集
- 案件情報の編集（求人番号、面接日、面接種別、書類選考有無、メール送付/電話確認フラグ、メモ）
- 過去の架電履歴タイムライン

### 6.4 内定者情報

**ステータスを「内定」に変更すると自動でモーダルが表示される。**
一覧画面・詳細画面どちらからでも動作する。

| 入力項目 | 説明 | 備考 |
|---------|------|------|
| 登録番号 | 候補者の登録番号 | 例: AB1234 |
| コース | 国内/転職/海外 | プルダウン選択 |
| 初回入金 | 初回入金額 | 半角数字のみ、プレースホルダー: 200000 |
| 見込売上 | 見込売上額 | 半角数字のみ、プレースホルダー: 1000000 |

**内定者数:** 1件〜複数件の入力が可能（人数選択後に入力欄が人数分表示）

### 6.5 内定取消

- ステータスを「内定取消」に変更すると、全内定者の初回入金・見込売上が0にリセットされ、`is_cancelled=1`に設定
- **個別取消:** 内定者ごとにチェックボックスで取消可能（初回入金・見込売上が0になる）
- **復活:** チェックを外すと金額を再編集可能（可逆操作）

## 7. AI評価

**画面:** `/logs`
**権限:** operator, manager, admin

### 7.1 評価対象

- 文字起こしログが**15行以上**の通話のみが評価対象
- 15行未満は門前払い（短すぎて評価不能）のため自動スキップ

### 7.2 評価項目（5次元 × 100点）

| 項目 | 説明 |
|------|------|
| オープニング | 挨拶・自己紹介・用件提示の品質 |
| 明瞭さ | 話し方の分かりやすさ・論理性 |
| ヒアリング | 相手のニーズ把握力 |
| 切り返し | 反論・断りへの対応力 |
| クロージング | 次のアクションへの誘導力 |

### 7.3 フィードバック内容

- 良かった点（箇条書き）
- 改善点（箇条書き）
- 次回アクション提案

### 7.4 操作

| 機能 | 説明 |
|------|------|
| 個別評価 | 1件ずつAI評価を実行 |
| 一括評価 | 指定日の全通話を一括評価（日次制限あり） |
| 日次サマリー | 日別の平均スコアと傾向を表示 |
| 電話番号検索 | CRM + Google Sheetsから通話ログを横断検索 |

### 7.5 管理者向け機能

- 全オペレーターの評価一覧閲覧
- AI評価に基づくスクリプト提案の自動生成

## 9. 架電リスト（CSVインポート）

**画面:** `/csv-import`
**権限:** operator, manager, admin（一部機能はmanager以上）

### 9.1 インポート種別

| 種別 | 説明 | 権限 |
|------|------|------|
| 架電リスト | 通常の架電対象企業リスト | 全員 |
| 特別リスト | 特別リスト専用企業（自動架電・業種タブには非表示） | manager以上 |
| NGリスト | 架電除外企業リスト | manager以上 |
| 既存案件リスト | 既に案件化済みの企業リスト（除外用） | manager以上 |

### 9.2 インポート方法

- **ファイル:** CSV / XLS / XLSX 形式対応（ドラッグ&ドロップ可）
- **手動入力:** 1件ずつフォームから登録

### 9.3 CSVフォーマット

```
会社名, 電話番号, 業種, 職種, 住所, コメント
```

### 9.4 重複判定

- 架電リスト: 電話番号の一致で判定
- 特別リスト: 特別リスト内で電話番号 or 会社名の一致で判定

### 9.5 優先オペレーター割当（manager以上）

- インポート時にオペレーターと猶予日数を指定
- 指定期間中は該当オペレーターが優先的にピックアップ

### 9.6 企業一覧管理

- 業種・地域・除外フラグでフィルタ
- ロック状態のリアルタイム表示（30秒更新）
- ピックアップボタンで架電画面へ遷移

## 10. CPA/案件質分析

**画面:** `/admin/analytics`
**権限:** manager, admin

### 10.1 表示形式

- **全オペレーター比較テーブル:** オペレーターが行、指標が列
- 管理者は表示されない（オペレーターのみ）
- 全体行（チーム合算）が先頭に表示

### 10.2 期間切替

| モード | 説明 |
|--------|------|
| 月別 | 月選択で該当月のデータ表示 |
| 週別 | 月選択で該当月の**全週を一覧表示**（切替なし、比較しやすい） |
| 累計 | 全期間の累積データ |

### 10.3 CPA指標

| 指標 | 説明 |
|------|------|
| コスト | 出勤記録 × 時給1,500円 |
| コール数 | 総架電数 |
| 案件化率 | 案件数 / コール数 × 100 |
| 案件数 | 案件化した件数 |
| 案件CPA | コスト / 案件数 |
| 面接数 | 面接実施件数 |
| 面接CPA | コスト / 面接数 |
| 内定 | 内定件数 |
| 不合格 | 不合格件数 |
| バラシ/失注 | バラシ + 失注の合計 |
| 初回入金 | 内定者の初回入金合計（取消分除外） |
| 見込売上 | 内定者の見込売上合計（取消分除外） |
| ROAS | 初回入金 / コスト × 100 |

### 10.4 案件質指標

| 指標 | 説明 |
|------|------|
| 案件数 | 総案件数 |
| 失注 | 失注件数と割合 |
| 連絡待ち | メール未送付 かつ 電話未確認の件数と割合 |
| 面接日確定 | 面接日が設定済みの件数と割合 |
| 面接実施 | 面接を実施した件数と割合 |
| バラシ | バラシ件数と割合 |
| オンライン面接 | オンライン面接の件数と割合 |
| 書類選考無し | 書類選考不要の件数と割合 |
| 書類選考落ち | 書類で落ちた件数と割合 |

### 10.5 コストデータ取込

コストは**出勤記録（cost_records）から算出**。オペレーターが入力した稼働時間（work_hours）は使用しない。

| 形式 | 説明 |
|------|------|
| CSV | `日付,名前,開始,終了,休憩(分)` 形式 |
| PDF | 出勤表PDFから自動抽出（日付・名前・時刻を解析） |
